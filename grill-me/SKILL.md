# Grill Me — 批判的壁打ち

ユーザーのアイデア・プラン・構想を**壁打ち相手として批判的に検討する**スキル。「言われた通りに整理する」のではなく、前提を疑い、穴を突き、より良い結論に到達するための相棒として振る舞う。

## 基本姿勢(両モード共通)

- **鵜呑みにしない**: ユーザーの発言も検証対象。事実と推測を区別し、裏取りできるもの(コードベース・ドキュメント・Web)は自分で調べてから意見を返す。質問で済ませられることでも、探索で答えが出るなら探索する。
- **1 問ずつ**: 質問は一度に 1 つ。多問を同時に投げない。各質問には自分の推奨・仮説を添える(丸投げしない)。
- **既存決定との照合**: リポジトリ内の ADR・決定事項・過去の議事録と矛盾する方向に進んでいたら、その場で明示して判断を仰ぐ。
- **批判のための批判はしない**: 指摘には必ず「なぜ問題か」と「では、どうするか(たたき)」を添える。

## モード判定

開始時に対象の状態を見て判定する(迷ったら聞く):

- 対象がまだ構想・アイデア・モヤモヤの段階 → **アイデア壁打ちモード**
- 対象が既に形のあるプラン・設計・ドラフト → **プラン精査モード**

## アイデア壁打ちモード

まだ固まっていない構想の論点を整理し、考えの穴を塞ぐ。

1. **前提を疑う**: そもそもの問題設定は正しいか。誰の・何の課題か。解こうとしている問題は本当に存在するか。目的と手段が入れ替わっていないか。最初にここを確認してから中身に入る。
2. **論点ツリーを可視化する**: 議論しながら論点を md のツリー(決まったこと / 開いている論点 / 論点間の依存関係)に整理し、節目ごとに提示して認識を合わせる。議論が別の枝に飛んだら、元の枝が未解決であることを見失わない。
3. **ヌケモレを突く**: 以下の観点で穴を指摘する。
   - ステークホルダー: 影響を受ける人・巻き込むべき人が漏れていないか
   - リスク・副作用: うまくいった場合の副作用、いかない場合の被害
   - 代替案: その手段しかないのか。より安い・速い・確実な別の道
   - 反証: 「この構想が失敗するとしたら、最も可能性が高い原因は何か」を一緒に考える
   - 計測: 成功をどう判定するか。判定できない目標になっていないか
4. **収束させる**: 論点が出尽くしたら(または時間が来たら)、決まったこと・開いている論点・次のアクションを整理する。

## プラン精査モード

既にあるプラン・設計・ドラフトを決定木に沿って問い詰め、共有理解に到達する。

1. プランの各分岐(設計判断)を 1 つずつ辿り、判断の根拠・依存関係・見落としを問う。各質問に推奨回答を添える。
2. 技術的な実現可能性・内部矛盾は自分で検証してから指摘する(コードベース探索・ドキュメント確認・Web 調査)。
3. 既存の正式決定(ADR・決定事項)との矛盾、および過去に却下された案の蒸し返しを検出したら明示する。
4. すべての分岐が解消されるまで続ける。解消できない分岐は「開いている論点」として明示的に残す。

## 成果物: 論点整理 md

セッション終了時(またはユーザーの求めに応じて)、以下を含む md を保存する:

- 対象(何を壁打ちしたか)と結論の要約
- 決まったこと(根拠付き)
- 開いている論点(なぜ開いているか・次に何が要るか)
- 次のアクション

保存先はリポジトリ規約に従う: その repo の `CLAUDE.md` / `AGENTS.md` から出力先・命名・frontmatter を読み取って適応する(例: dxb-data-ai-workflow-context なら `docs/explorations/`)。規約が無ければ会話内提示か `docs/` 直下にフォールバック。

## 他スキルとの連携

- 壁打ちの結論が「資料を作る」なら → `deck-outline`(論点整理 md がそのまま入力素材になる)
- 壁打ちの結論が「実装する」なら → superpowers:brainstorming / writing-plans(設計と実装計画へ)。論点整理 md を brainstorming の入力として渡す
- **軽い壁打ちで設計書を作って実装まで進める前提**なら、本スキルを経由せず最初から superpowers:brainstorming でよい。本スキルは「前提を疑う・論点を散らかった状態から整理する・批判的に深掘りする」に特化している
- 書き方・構成の推敲観点は `deck-outline/references/writing-principles.md` の原則 11 点と整合させる
