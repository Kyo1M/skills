# 画像生成プロンプト集（continova v1）

codex CLI で生成するスライド用ビジュアルのプロンプトテンプレート。すべて continova v1 デザインシステムに揃える。ワークフローは `image-generation.md` を参照。

> **注意**: PowerPoint 風の青/グレー/緑配色テンプレート（`article-pipeline` 等）を流用しない。continova v1 は白地・Klein Blue 単一アクセント・フラット・余白が要。

## 共通スタイルブロック

すべてのプロンプトの先頭にこれを置く。

```
【日本語】
スタイル: ミニマルなコンサルティング資料のイラスト、フラットなアイソメトリック、
  幾何学的でクリーンな形
背景: 純白
配色: アクセントは Klein Blue #002FA7 のみ。それ以外はインク #111111 と
  グレー #6F7175、淡い面は #EAF1FF / #F6F5F2
余白: たっぷり取る。1つの主題に絞る
禁止: 光沢のある3Dレンダ、グラデーション、ドロップシャドウ、ストック写真風、
  紫系コーポレートクリシェ、ノイジーな背景、装飾ブロブ
テキスト: 入れない（または英単語ラベル数語まで）。日本語の文章は焼き込まない
```

```
【English】
Style: minimal consulting-deck illustration, flat isometric, clean geometric shapes
Background: pure white
Color: single accent Klein Blue #002FA7 only; everything else ink #111111 and
  grey #6F7175; light fills #EAF1FF / #F6F5F2
Layout: generous whitespace, one dominant idea
Avoid: glossy 3D render, gradients, drop shadows, stock-photo look,
  purple corporate cliche, noisy background, decorative blobs
Text: none (or a few English-word labels at most); never bake in Japanese sentences
```

## テンプレート A: ヒーロー / セクション扉イラスト

タイトルスライドやセクション区切りに置く、デッキ主題を象徴するアイソメトリックイラスト。

```
<共通スタイルブロック>

主題: <デッキ/セクションの主題を1語〜1フレーズで。例: データ基盤の構築、
  実験による意思決定>
構図: 中央〜右寄りに1つのアイソメトリックなシーン。左側はテキストを置くため
  大きく余白を空ける
要素: 主題を表す抽象的な構造物（積層・経路・ノードなど）を幾何学的に
表現:  Klein Blue は主役要素1〜2箇所だけ。残りはインクとグレーの線画＋淡い面
アスペクト比: 16:9
```

HTML 側: `.hero-visual` に配置し、タイトル・eyebrow・トップメッセージは HTML テキストで重ねる。

## テンプレート B: アイソメトリック概念図解（主役オブジェクト）

スライドの主役オブジェクトを、構造・流れを直感的に見せるアイソメトリック図解にする。

```
<共通スタイルブロック>

図解する構造: <システム/プロセス/関係を説明。例: 3層のデータパイプライン、
  人とAIの協働ループ>
構図: アイソメトリックな図。ノード/ブロック/経路を幾何学的に配置
ラベル: 焼き込まない。各ノードの位置が後から HTML テキストで指せるよう、
  ノードは明確に分離し、周囲に余白を残す
強調: 主役ノードは Klein Blue を輪郭・上面の一部・コネクタに効かせる。
  面全体のベタ塗りは避け、青の視覚面積は控えめに。残りはインク／グレーの線画
アスペクト比: 横長（スライド本文の主役領域に合わせる）
```

HTML 側: `.figure` に配置。ノードのラベル・数値・凡例は HTML テキストで重ねる/隣置きする（`slide-patterns.md` Pattern 12）。

## テンプレート C: アイコンセット（1 セッションで一括生成）

カード見出し・eyebrow 横などに添えるアイコン。**1 回の `codex exec` で必要分をまとめて生成**し、画風を揃える。

### アイコン共通スタイル仕様

```
各アイコン:
- 正方形キャンバス、純白背景
- フラットでミニマルな2トーン（インク #111111 ＋ Klein Blue #002FA7）
- シンプルな幾何形。線の太さ・塗りの密度をすべて統一
- 中央配置、周囲に均等な余白
- 光沢・グラデーション・影なし。絵文字調にしない
- アイソメトリックではなく正面のフラットアイコン（図解と役割を分ける）
```

### 推奨アイコンと意味

デッキで実際に使うものだけ生成する。意味に対する名前（`<name>.png`）を固定し、見た目が変わっても名前は不変にする。

| name | 意味 |
|---|---|
| `target` | 目的・ゴール |
| `cycle` | 仕組み・循環・PDCA |
| `layers` | 階層・レイヤー構造 |
| `map-pin` | フェーズ・マイルストーン |
| `wall` | 壁・障壁・制約 |
| `play-circle` | デモ・実演 |
| `flask` | 試行・実験・PoC |
| `tool` | ツール・実装 |
| `sparkle` | AI機能・自動化 |
| `git-branch` | 分岐・選択肢 |
| `message-square` | 議論・対話 |
| `alert-triangle` | 注意・リスク |
| `check-circle` | 達成・完了 |
| `database` | データ・資産 |

色: 主役・推奨は Klein Blue を効かせる。標準はインク、脇役はグレー（`design-system.md`「アイコンの使い方」参照）。
