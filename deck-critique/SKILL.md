# Deck Critique — 資料の批評・推敲

構成 md や完成 HTML スライドを、**作成した文脈から離れた別の目で検査する**スキル。作った本人(同じ会話)のセルフチェックは自分の判断を追認しがちなので、批評だけを独立の工程として行う。

## 実行方法

- **可能ならサブエージェント(新しいコンテキスト)で実行する**。メイン会話で資料を作った直後なら、Task/Agent ツールで批評専用のエージェントを起動し、資料ファイルと(あれば)元素材だけを渡す。作成時の経緯・言い訳を持ち込ませないため。
- サブエージェントが使えない環境では、そのまま本スキルの検査手順を実行する(それでも工程を分ける価値はある)。

## 入力

1. **資料本体**(必須): 構成 md または HTML スライド
2. **元素材**(任意): ユーザーのアイデアメモ、議事録、決定事項、deck-outline のトレーサビリティ表。渡されれば脱落突合(検査 4)を行う
3. **読者と場**(任意): 誰向けか、どの会議で使うか。不明なら資料から推定し、推定であることを明示する

## 検査項目(この順で)

### 1. ストーリーの一貫性

- 資料の「狙い」を**本文から逆算で 1 文抽出**する(作成者の意図ではなく、書かれたものから読み取れる主張)。抽出できなければそれ自体が最重要の指摘。
- 各ページの主張が狙いにぶら下がっているかを 1 枚ずつ確認する。ぶら下がらないページ、唐突な対比軸、前のページと接続しない飛躍を指摘する。
- 背景→課題→解決→具体の因果が一本で通っているか。

### 2. 日本語の自然さ

`deck-outline/references/writing-principles.md` の原則(特に 7)に照らして全ページを検査する:

- タイトル: 体言止めか。動詞締め・比喩ジャーゴン(「背骨」等)・抽象的な体言化(「〜という到達点」)がないか
- リード文(トップメッセージ): 短文の連打になっていないか。結論を言っているか。改まった語(「本日」等)か
- ボディ: 体言(単語)の羅列がないか。「だから」の多段連鎖がないか
- 全体: 読者の語彙か(専門語・カタカナ抽象語の混入)。声に出して読んで、実際に打ち合わせで人が言う言い回しか

### 3. 議論のしやすさ

- 【議論】ページは 1 ページ 1 決定か。問い→方針(たたき)→この場で決めたいこと、の型か
- 方針のたたきに、その場で妥当性を判断できるだけの具体があるか(抽象ラベルへの圧縮が起きていないか)
- 選択肢 A/B/C の並列比較を無許可で持ち込んでいないか(既定は方針の直接提示)
- 「まとめ」ページがあるか。振り返りと持ち帰りが 1 枚に畳まれているか
- 想定時間に対してページ数・議論数が現実的か

### 4. 脱落の突合(元素材があれば)

- 元素材の項目を番号付きで列挙し、各項目が資料のどのページに反映されたかを突合する
- 反映が見つからない項目を「脱落候補」として列挙する(意図的な省略かの判断は作成者・依頼者に委ねる)
- 元素材の具体(数値・チェック項目・固有名詞)が抽象ラベルに縮んでいる箇所を指摘する

### 5. 完結性・その他

- 資料単体で理解できるか(前版差分・社内文脈への暗黙依存がないか)
- TBD・プレースホルダ・空欄の残存
- 事実の誤り・数値の不整合(裏取りできるものは自分で確認する)

## 出力フォーマット

**勝手に修正しない。** 以下の形式で指摘リストを返し、修正の取捨選択は依頼者に委ねる:

```
## 総評(2〜3 文: 狙いの逆算結果と、最も効く改善 1 点)

## 指摘(重要度順)
1. [重大|中|軽微] <ページ#> <指摘の一文>
   - なぜ問題か: ...
   - 修正案: <実文言のたたき>

## 脱落候補(元素材があれば)
- <素材項目> → 反映先が見つからない(意図的なら OK)

## 良い点(2〜3 点、維持すべき判断)
```

指摘には必ず**実文言の修正案**を添える(「自然にする」のような抽象指示で終えない)。重大 = ストーリーが壊れている・議論が空転する / 中 = 読者が引っかかる / 軽微 = 磨き。

## 他スキルとの関係

- `deck-outline`(構成設計)→ `html-slide-deck` / Claude Design(実装)→ **本スキル(批評)** の順で使うのが基本線。構成段階(md)と実装後(HTML)のどちらにもかけられる。
- 批評の観点の正本は `deck-outline/references/writing-principles.md`。原則が更新されたら本スキルの検査は自動的にそれに従う(原則をコピーせず参照する)。
